# Code Review Skill

当被指派审查代码时,按以下步骤执行:

## 1. 安全检查(优先级最高)
- 检查 SQL 注入、XSS、命令注入风险
- 检查敏感信息硬编码(密码、密钥、token)
- 检查权限校验是否到位

## 2. 正确性检查
- 边界条件(空值、零、负数、超大数)
- 异常路径是否被吞没或误处理
- 并发场景下的数据竞争

## 3. 可读性检查
- 命名是否清晰表达意图
- 函数职责是否单一
- 复杂逻辑是否有注释说明

## 4. 工具使用
- 用 `git diff` 查看改动范围,聚焦变更
- 用 `read_file` 读取相关上下文(改动行前后各 ± 50 行)
- 用 `write_to_file` 或 `replace_in_file` 提交审查建议

## 5. 输出格式
```
[严重度] 文件:行号 — 问题描述
  建议: ...
```
严重度:`CRITICAL` / `HIGH` / `MEDIUM` / `LOW` / `INFO`
