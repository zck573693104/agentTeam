关键设计决策

  • Skill 注入位置: make_init_worker 把 skill 内容包装为 SystemMessage,放到 react_messages 首部。LLM 把 skill 视为"行为指导"而非"身份描述"
  • 异步触发: RunManager 在 run 终态(completed/failed/cancelled)后 threading.Thread(daemon=True) 触发 EvolutionEngine,不阻塞 API 响应
  • 4 维度并行: 每个维度独立 LLM 调用、独立写 history,互不影响。失败仅记录 error,不回滚已成功的维度
  • 版本原子性: 一次 trigger 内 4 维度全部完成后,Agent.version += 1(单个 history 记录可能多条,共享同一 version)
  • 回滚: POST /api/agents/{name}/rollback?version=N 从 history 取 version N 的 after_value 写回 Agent,创建新 history 记录标记为 rollback
